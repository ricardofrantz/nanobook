## Commands to compute release numbers

Run this after sanitized audit excerpts exist:

```bash
python3 examples/paper-live-ibkr/soak_metrics.py examples/paper-live-ibkr/sanitized-audit.jsonl
```

The command emits JSON with calendar days, first/last day, rebalance count, order/fill/cancel counts, reconcile count, reconcile mismatch count, incident count, manual intervention count, and a compact incident event list. Copy those values into the README Battle-tested source table above before editing README.

## Close criteria for bd-3mc

Do not close the README/release bead until all are true:

- README contains a Battle-tested section with all numeric fields filled from the source table above.
- CHANGELOG has a v0.15 entry describing the actual paper-live evidence.
- Version bumps are reviewed against current repository versions and documented in the versioning decision log.
- Release tag is created and pushed, or explicitly deferred with a bead/comment explaining why.
- `python/uv.lock` or any other unrelated generated file is not accidentally bundled into the release commit.
